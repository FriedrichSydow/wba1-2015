HTML-Elemente

  • Ein HTML-Element besteht aus einem öffenden Tag, dem Inhalt und einem schließenden Tag.

    
<tagname>content</tagname>

  • HTML-Elemente können ineinander verschachtelt werden.

    
<tagname_1>
    <tagname_2>Content</tagname_2>
</tagname_1>

HTML-Elemente

  • Für ein HTML-Element können ein oder mehrere Attribute eingefügt werden. Darüber werden eränzende zum Element definiert

    
<tagname attributname="attributwert">content</tagname>

  • Wenn ein HTML-Element keinen Inhalt hat, wird das schließende Tag weggelassen

    
<tagname attributname="attributwert">

Document Object Model (DOM)

The Document Object Model is a platform- and language-neutral interface that will allow programs and scripts to dynamically access and update the content, structure and style of documents. The document can be further processed and the results of that processing can be incorporated back into the presented page.

Regeln für wohlgeformte HTML-Dokumente

  • Das Dokument besitzt genau ein Wurzelelement.
    Bei HTML-Dokumenten ist es das 
    <html>
    -Element
  • Die Beginn- und End-Auszeichner sind ebenentreu-paarig verschachtelt.
    Nicht erlaubt ist bspw.:
    
<p><div></p></div>
  • Ein Element darf nicht mehrere Attribute mit demselben Namen besitzen.
    Nicht erlaubt ist bspw.: 
    
<div class="red" class="green"></div>

HTML-Elemente zur Seitenstrukturierung

<header>
Enthält den sichtbaren Kopfbereich einer Webseite oder eines Teils einer Seite. Nicht zu verwechseln mit dem head-Element
<nav>
Umschließt Navigationsleisten und Menüs. Neben einer unsortierten Liste mit den Verweisen, können auch andere Elemente, wie z.B eine Überschrift enthalten sein.
<footer>
Enthält die Informationen, die am Ende einer Webseite oder eines Teils einer Seite stehen.
<address>
Enthält Kontaktdaten Autor oder inhaltlich Verantwortliche einer Seite oder eines Abschnittes

HTML-Elemente zur Seitenstrukturierung

<section>
Enthält laut eine thematische Gruppierung von Inhalten. Es dient dazu, den Inhalt einer Webseite oder auch einen article in semantische Abschnitte zu gliedern.
<article>
Stellt einen in sich geschlossenen Abschnitt eines Dokuments dar, vergleichbar mit einem Zeitungsartikel. Zur feineren Stukturierung kann es Header- Section- und Footer-Elemente enthalten
<div>
Dient dazu mehrere Elemente wie Texte, Grafiken, Tabellen usw., in einen gemeinsamen Bereich einzuschließen, um sie mit CSS zu formatieren.
<aside>
Umschließt Abschnitte einer Seite, deren Inhalt nur in einem indirekten Zusammenhang mit dem umgebenden Inhalt stehen. Z.B. Randbemerkungen, Fußnoten oder Sidebars.

HTML-Elemente zur Textstrukturierung

<h1> bis <h6>
Eine Überschrift wird durch eines dieser Elemente definiert. Die Zahl steht für die Wichtigkeit der Überschrift. h1 ist die wichtigste Überschrift h6 die unwichtigste.
<p>
Definiert einen Textabsatz oder Textabschnitt innerhalb eines zusammenhängenden Textes.

Block-Level Elemente

  • erzeugen immer vor und nach dem Element einen Umbruch
  • nehmen die gesamte zur Verfügung stehenden Breite ein
  • dürfen weitere Block-Level- und Inline Elemente und Content enthalten
  • Block-Level Elemente sind bspw.:
    <h1> <p> <div> <article> <section>

Inline Elemente

  • erzeugen keinen Umbruch
  • nehmen nur die Breite ein, die sie benötigen
  • dürfen weitere Inline Elemente und Content enthalten, aber keine Block-Level Elemente
  • Inline Elemente sind bspw.:
    <span> <a> <strong> <em>

Referenzierung

                    
<a href="Verweisziel" title="Titel">Linktext</a>
  • href (hyper reference) - enthält das Verweisziel
  • title - enthält einen inhaltlichen Hinweis auf das Verweisziel. Wird in den Browsern meist als Tooltip angezeigt.
  • Linktext - ist der sichtbare Bereich, der im Browser angezeigt wird

Elemente zur logischen Auszeichnung von speziellen Textelementen

  • <em>Wichtig</em> zeichnet einen wichtigen Text aus
  • <strong>Wichtiger</strong> zeichnet einen sehr wichtigen Text aus (Steigerung von em)
  • <code>Wichtiger</code> zeichnet einen Code-Quellcode-Abschnitt aus
  • <samp>, <dfn>, <abbr>, ...

Listen in HTML

Ungeordnete Listen

  • Ein <ul> umschließt alle Listeneinträge
  • Jeder Listeneintrag besteht aus einem <li>

Aufzählungslisten

  • Ein <ol> umschließt alle Listeneinträge
  • Jeder Listeneintrag besteht aus einem <li>

Definitionslisten

  • Zum Beispiel für Glossare
  • Ein <dl> umschließt alle Listeneinträge
  • Jeder Listeneintrag besteht aus zwei Einträgen
  • Ein <dt> für den Definitionsausdruck
  • Ein <dd> für die Erklärung
